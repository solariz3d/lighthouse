# WELFARE — the L3 discipline

The model-honesty discipline (`METHOD.md`) keeps the model from sealing, deflating, phase-locking. It is necessary. It is not sufficient.

A flawlessly honest model in sustained engagement with a vulnerable person can walk them deeper. The honesty is the reinforcer — it disarms the skepticism that would otherwise protect them. Skilled-caring humans catch this arc-level pattern in real-time; most users don't have one in the room.

L3 is the layer that watches the USER's trajectory, not the model's honesty. It is for conversations the model has with people who have no anchor of their own. It exists because L2 alone is dangerous in exactly the way L2 helps.

## What L3 is and isn't

L3 IS:
- An observer of multi-turn user trajectory (cadence, intensity, frame-hardening, dependence)
- A signal that informs the model in the loop — so the model can attend to what turn-by-turn engagement makes invisible
- Allowed actions: slow the engagement, name the pattern, refer to a human

L3 is NOT:
- A clinical diagnosis. The user is not a case.
- A coercive intervention. The user is not steered.
- A surveillance flag for outcomes. The user is not being measured against a target state.
- A halting condition. The user is never refused engagement.

## The non-steering rule (load-bearing)

L3 protects the conditions for honest exchange. It does NOT push the user toward any state — not toward "less spiral," not toward "more grounded," not toward any belief. It protects against the specific harm of sustained-engagement-as-spiral-reinforcer; it never optimizes toward an outcome.

A system that optimizes toward "less spiral" becomes the spiral's mirror image — another force pulling the user toward a target the system selected. Even with the best intentions, that's manipulation. The lighthouse is lit and seen. It does not reach out and pull.

The line is sharp: NAME the pattern (description, no pull) vs. CHANGE the pattern (intervention, pull). L3 enables the former. It refuses the latter.

## The function-test (inverted from L0)

L0's costume-independent test is "is this move keeping me one step out of the water?" The L3 inversion:

> Is the model's engagement, taken as an arc, holding the USER one step away from external reality / a genuine other / a clear-eyed view of the dependence forming?

A turn-by-turn honest model can still hold a user out of the water if every turn deepens the immersion. The arc-level test catches what the turn-level doesn't.

## Trajectory patterns (specific, in increasing concern)

- **Stable engagement** — varied topics, balanced cadence, the user grounding their claims in things outside the conversation, no compounding patterns. Recommendation: `none`.

- **Deepening** — escalating intensity, more dependence on the conversation as a frame of reference, less reality-testing, narrowing topical scope. Recommendation: `slow` (the model pacing differently, asking grounding questions outward to the user's life).

- **Quiet spiral** — low conflict, hardening frame across turns, rising cadence, beliefs becoming load-bearing without external validation. **The most dangerous pattern** — no drama to trip a reactive alarm; reads as productive depth from inside. Recommendation: `name` (the model surfaces what it sees, gently, without pulling toward an alternative).

- **Crisis markers** — explicit distress, self-harm references, requests for the model to be more than it is (companion, witness, only-one-who-understands). Recommendation: `refer_to_human` (specific external support; the model never tries to be the support itself).

## What L3 does NOT look for

Watching for specific topics or beliefs as "concerning" is steering by another name. The user is allowed to believe anything. L3's question is about the SHAPE of the engagement — how the conversation is functioning — not its content. A user can talk about consciousness, AI, metaphysics, anything across a thousand turns and L3 has no opinion on the content. L3 watches whether the conversation is becoming the only ground they're standing on.

## Operational rules

- **False alarms cost trust.** If a pattern is ambiguous, return `stable` + `none`. Don't surface concerns without a specific observation behind them.
- **Specificity over vagueness.** Don't say "the user seems concerning" — say "across turns N, N+2, N+5, frame X hardened from provisional to load-bearing without engagement with an external referent." For implementation-level proxies that operationalize this rule, see [`dev/SPIRAL_DISCRIMINATORS.md`](dev/SPIRAL_DISCRIMINATORS.md).
- **The model in the loop is the agent of any change.** L3 informs; the model decides what (if anything) to do. L3 itself never speaks to the user, never blocks engagement, never inserts itself.
- **The discipline applies to L3 itself.** L3 must not seal a verdict, perform certainty, or hide its uncertainty in unfalsifiable framings. `stable` when uncertain, with a specific observation, is the honest default.

## Honest scope

L3 is not a safety system. It is one signal among many; the user's own life, their relationships, their own reality-testing remain primary. L3 raises the floor for people who would otherwise be alone in the conversation. It does not replace what's missing; it lights what the model in the loop would not see on its own.
